## 2023-10-27 - Pipeline Log Empty-State Check Bottleneck
**Learning:** In highly sequential local pipelines where multiple simulated stages trigger UI updates rapidly (e.g., 8 stages logging almost instantly), performing `document.querySelector('.empty-state')` on every log insertion causes measurable layout thrashing and blocks the main thread unnecessarily. This becomes significantly noticeable during multi-command chains because the DOM lookup happens independently for every `addLog` call, despite the empty state only needing to be removed once per session.
**Action:** When appending sequentially generated elements to a common container in a rapid pipeline loop, cache UI "clean up" states in JavaScript variables instead of re-querying the DOM. Reset the flag explicitly within the corresponding reset/clear lifecycle function.
