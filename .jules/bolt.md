## 2025-02-24 - Canvas Animation Loop String Interpolation Causes GC Pressure
**Learning:** In standard frontend canvas animations inside `requestAnimationFrame`, performing string interpolation to construct colors dynamically on every frame (like `` rgba(255,255,255,${alpha}) `` inside a loop iterating over hundreds of items) generates an immense amount of small, short-lived string allocations. This creates noticeable GC churn leading to micro-stutters. Similarly, re-allocating a `new Uint8Array(analyser.frequencyBinCount)` on every frame creates unnecessary typed array allocations.
**Action:** Always pre-allocate fixed-size array buffers outside the animation loop, and reuse the same instance (by mutating its contents, like calling `analyser.getByteFrequencyData` with a pre-allocated array). For alpha-blending in canvas loops, define a constant `fillStyle` and mutate the context's `globalAlpha` instead of interpolating strings. For static colors, pre-compute them into an array.
