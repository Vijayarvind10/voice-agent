## 2026-04-05 - Animation Loop Optimizations
**Learning:** Browsers do not throttle `requestAnimationFrame` for elements hidden via `display: none` or positioned off-screen when the tab remains active. In this codebase, the background `#starCanvas` was hidden by default but its animation loop (`drawStars()`) was still consuming CPU 60 times a second. Additionally, the audio visualizer (`drawVis()`) was allocating a new `Uint8Array` on every frame, generating significant garbage collection (GC) pressure.
**Action:** Use an `IntersectionObserver` to track the visibility of hidden canvas elements and explicitly pause their animation loops using `cancelAnimationFrame()`. For high-frequency loops like `drawVis()`, allocate the buffer (`Uint8Array`) once when the loop starts and reuse it across frames to mitigate GC pressure.
