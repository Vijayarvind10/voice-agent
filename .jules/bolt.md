## 2024-05-18 - Animation Loop Garbage Collection Pressure
**Learning:** In the frontend (`index.html`) canvas animation loops like `drawStars` and `drawVis`, using dynamic string interpolation for colors (e.g., `` `rgba(255,255,255,${s.a * 0.4})` ``) inside a `requestAnimationFrame` callback forces the JS engine to allocate new strings 60 times per second, multiplied by the number of particles/iterations (e.g. 120 stars). This causes significant garbage collection pressure, leading to frame drops or micro-stutters. A similar issue occurs if typed arrays (e.g., `new Uint8Array(analyser.frequencyBinCount)`) are allocated inside the render loop rather than being reused.
**Action:** When working on rendering or animation loops, hoist memory allocations (arrays, objects) outside the loop and reuse them. For canvas coloring, use constant color strings (e.g., `#ffffff`) and mutate context properties like `globalAlpha` instead of generating new CSS color strings every frame.
