## 2024-05-19 - Canvas Animation Garbage Collection
**Learning:** In high-frequency animation loops like `requestAnimationFrame` for canvas rendering (e.g., `drawVis`, `drawStars`), dynamically constructing string interpolation colors (like `rgba()`) and creating arrays (e.g., `new Uint8Array`) on every frame creates substantial garbage collection pressure. This leads to observable micro-stutters and frame drops in the UI due to the constant memory allocation and deallocation cycle.
**Action:** Always hoist array instantiations (like `Uint8Array`) out of animation loops and reuse them. Replace dynamic `rgba()` string interpolation with constant precomputed colors (e.g., an array of strings or using `globalAlpha` with a constant color) to prevent rapid string allocation during `requestAnimationFrame`.
