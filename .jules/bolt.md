## 2026-04-07 - [Frontend Performance: CPU Burn from Hidden Canvas Elements]
**Learning:** Browsers do not throttle `requestAnimationFrame` for elements hidden via `display: none` or moved off-screen if the browser tab itself remains active. In `index.html`, `#starCanvas` was running a 60fps animation loop calculating math for 120 elements despite being hidden by default (`display: none` in CSS), resulting in unnecessary CPU usage and battery drain. This is a critical edge case in frontend architecture when using hidden legacy layers or hidden visual effects.
**Action:** When implementing or encountering continuous animation loops (especially using `<canvas>` and `requestAnimationFrame`), ALWAYS wrap them with an `IntersectionObserver`. Explicitly use `cancelAnimationFrame` when the element is not intersecting or is hidden (`getComputedStyle(element).display === 'none'`) to stop the loop, and request a new frame when it becomes visible again.
